💡 解説
オーバーロード解決では引数型の厳密一致が最優先される。int リテラル 10 は int 型なので、int 版が選ばれる。long への自動昇格は厳密一致がない場合のみ適用される。

問題 2: オーバーロード（boxing vs varargs）

}

}

✗
C
コンパイルエラー

💡 解説
オーバーロードの優先順位は、厳密一致 > 拡大変換 > オートボクシング > 可変長引数 の順。int リテラル 1 は Integer へのオートボクシングが可変長引数より優先されるため、Integer 版が呼ばれる。

問題 5: 引数の評価順序

int i=1;
int r = add(i, i++);
// add は (a+b)
System.out.println(r);
A
2
✓
B
3
✗
C
未定義
✗
D
コンパイルエラー
✗
💡 解説
引数の評価は左から右へ順に行われる。i++ は後置インクリメントなので、現在の値（1）を返してから i が 2 になる。したがって add(1, 1) が実行され、結果は 2。

問題 6: 同名ローカルとフィールド

✗

✓
C
コンパイルエラー
✗

💡 解説

問題 7: 可変長引数

A
0
✓
B
1
✗
C
コンパイルエラー
✗
D

✗
💡 解説
可変長引数 int... は内部的に int[] 配列として扱われる。引数を 0 個で呼び出すと、長さ 0 の配列が生成される。したがって a.length は 0 を出力する。

問題 9: private メソッドのオーバーライド？

class A{
  private void f(){}
}
class B extends A{
  void f(){}
}
A
オーバーライド
✗
B
別メソッド（隠蔽）
✓
C
コンパイルエラー
✗
D
実行時例外
✗
💡 解説
private メソッドは継承されないため、サブクラスで同名のメソッドを定義してもオーバーライドにはならない。B クラスの f() は A クラスの f() とは完全に独立した別のメソッドとして扱われる。
